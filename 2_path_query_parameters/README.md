## Path Parameter
Used to identify a specific resource.

Example:
GET /students/5

## Query Parameter
Used to filter or paginate data.

Example:
GET /students?skip=0&limit=10

## Key Differences
- Path parameters are mandatory
- Query parameters are optional

## Path Parameter
- Part of URL, used to identify specific resource.

## Query Parameter
- Comes after ?, used for filtering, pagination
